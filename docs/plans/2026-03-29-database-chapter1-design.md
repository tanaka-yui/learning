### docs/normalization.md

| セクション | 内容 |
|-----------|------|
| 1. 正規化とは | 目的、メリット・デメリット |
| 2. 正規形の一覧 | 各正規形の概要表 |
| 3. 第1正規形（1NF） | 定義、違反例→修正後SQL |
| 4. 第2正規形（2NF） | 定義（部分関数従属の排除）、違反例→修正後SQL |
| 5. 第3正規形（3NF） | 定義（推移関数従属の排除）、違反例→修正後SQL |
| 6. BCNF | 定義と3NFとの違い、例 |
| 7. 第4〜第7正規形 | 4NF/5NF/6NF/DKNF/7NF（概念説明中心） |
| 8. 非正規化 | 目的、メリット・デメリット、適用場面 |

**方針:** 1NF〜3NFは違反例・修正後のSQLを丁寧に示す。4NF以降は概念説明に留める。

### docs/exercise.md

テーマ: **スーパーのレジシステム**

要件:
- レシートには複数の商品が含まれる
- 商品にはカテゴリがある
- 会員カードで顧客を識別できる
- 担当したレジ係を記録する

構成:
| セクション | 内容 |
|-----------|------|
| 問題 | 要件提示（読者が設計を考える） |
| Step 0 | 非正規形テーブル + 発生する異常（更新/挿入/削除） |
| Step 1 | 1NFへの変換 + 残存する問題点 |
| Step 2 | 2NFへの変換 + 残存する問題点 |
| Step 3 | 3NFへの変換（最終形） |
| 解答まとめ | 最終ER図（テキスト表現）+ 各テーブルの役割 |

**方針:** 各Stepで変換後SQLと「なぜこの変換が必要か」をセットで示す。問題文を先出しして読者が自分で考えられる構成にする。
