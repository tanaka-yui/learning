# Webアプリケーションセキュリティ学習 - 概要

## OWASP Top 10とは

OWASP (Open Worldwide Application Security Project) Top 10は、Webアプリケーションにおける最も重大なセキュリティリスクをまとめたランキングである。OWASPコミュニティが定期的に更新しており、開発者やセキュリティ担当者がWebアプリケーションの脆弱性を理解し、対策を講じるための業界標準的なガイドラインとして広く活用されている。

## 本モジュールの目的

本学習モジュールでは、OWASP Top 10に含まれる代表的な攻撃手法を実際に体験し、その仕組みと対策方法を学ぶ。各攻撃タイプについて、意図的に脆弱性を持たせたコード（脆弱版）と、適切な対策を施したコード（対策版）の両方を用意し、比較しながら学習できる構成としている。

## 脆弱版 vs 対策版パターン

各攻撃タイプには、以下の2つのバックエンドサーバーが用意されている。

- **脆弱版 (vulnerable)**: 意図的にセキュリティ上の欠陥を含むコード。攻撃がどのように成立するかを確認できる。
- **対策版 (secure)**: 適切なセキュリティ対策を実装したコード。同じ攻撃が防がれることを確認できる。

また、それぞれに対応するフロントエンドを用意しており、ブラウザから操作しながら攻撃と防御の違いを視覚的に確認できる。

## 対象とする攻撃タイプ

### SQLインジェクション (sql-injection)

ユーザー入力をSQL文にそのまま埋め込むことで、意図しないSQL文を実行させる攻撃。データベースの情報漏洩、改ざん、削除などが可能になる。対策としてプリペアドステートメントやパラメータバインディングを使用する。

### クロスサイトスクリプティング - XSS (xss)

悪意のあるスクリプトをWebページに注入し、他のユーザーのブラウザ上で実行させる攻撃。Cookie窃取やセッションハイジャックなどが可能になる。対策として出力エスケープやContent Security Policyを使用する。

### クロスサイトリクエストフォージェリ - CSRF (csrf)

ユーザーが認証済みの状態を悪用し、意図しないリクエストを送信させる攻撃。ユーザーの権限で不正な操作が実行される。対策としてCSRFトークンやSameSite Cookieを使用する。

### 認証の不備 (auth-bypass)

認証・認可の実装不備を突いて、本来アクセスできないリソースや機能にアクセスする攻撃。脆弱なセッション管理、不十分なアクセス制御などが原因となる。対策としてJWTの適切な検証やロールベースアクセス制御を使用する。

### パストラバーサル (path-traversal)

ファイルパスの入力を操作し、本来アクセスできないディレクトリやファイルを読み取る攻撃。`../` のような相対パス指定でサーバー上の任意のファイルにアクセスされる。対策としてパスの正規化やホワイトリスト方式のファイルアクセス制御を使用する。

### コマンドインジェクション (command-injection)

ユーザー入力をOSコマンドの一部としてそのまま使用することで、任意のコマンドを実行させる攻撃。サーバーの完全な制御権を奪取される可能性がある。対策としてコマンド実行を避けるか、入力の厳密なバリデーションを使用する。

## デモの使い方

### 起動

各攻撃タイプは Docker Compose のプロファイルとして定義されており、Makefileから個別に起動できる。

```bash
# SQLインジェクションのデモを起動
make sql-injection

# 全攻撃タイプのデモを一括起動
make all
```

### 確認

起動後、以下のポートでアクセスできる。

| 攻撃タイプ | 脆弱版フロントエンド | 対策版フロントエンド |
|-----------|-------------------|-------------------|
| sql-injection | http://localhost:3000 | http://localhost:3001 |
| xss | http://localhost:3002 | http://localhost:3003 |
| csrf | http://localhost:3004 | http://localhost:3005 |
| auth-bypass | http://localhost:3006 | http://localhost:3007 |
| path-traversal | http://localhost:3008 | http://localhost:3009 |
| command-injection | http://localhost:3010 | http://localhost:3011 |

### 停止

```bash
make down
```

### 学習の進め方

1. まず脆弱版のフロントエンドにアクセスし、攻撃が成立することを確認する
2. 脆弱版のバックエンドコードを読み、なぜ攻撃が成立するのかを理解する
3. 対策版のフロントエンドで同じ攻撃を試み、防がれることを確認する
4. 対策版のバックエンドコードを読み、どのような対策が施されているかを理解する
