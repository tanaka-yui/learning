# UI追加 設計ドキュメント

## 概要

5つの Agent フレームワーク実装それぞれに `backend/` と `frontend/` ディレクトリを追加する。
既存コードは `backend/` に移動し、新規に React + Vite のチャット UI を `frontend/` として追加する。

## アーキテクチャ

### ディレクトリ構造（全フレームワーク共通パターン）

```
{framework}/
├── backend/         ← 既存コードをそのまま移動
│   ├── src/
│   ├── Dockerfile
│   ├── package.json (or pyproject.toml)
│   └── ...
└── frontend/        ← 新規 React + Vite チャット UI
    ├── src/
    │   ├── App.tsx
    │   └── main.tsx
    ├── Dockerfile
    ├── package.json
    └── vite.config.ts
```

## フロントエンド

### UI仕様

全フレームワーク共通の画面構成：

- **ヘッダー**: フレームワーク名 + フレームワーク固有カラー
- **sessionId**: ページロード時に UUID を自動生成、ヘッダーに表示
- **チャット表示**: user / assistant メッセージを吹き出しで表示
- **入力欄**: テキストボックス + 送信ボタン（Enter 送信対応）
- **UX**: 送信中はボタンを無効化

### API呼び出し方針

| フレームワーク | 実装 |
|---|---|
| mastra | `@mastra/client-js` → `client.getAgent('task-agent').generate({ messages, resourceId, threadId })` |
| mastra-fastify / strands-* / claude-agent-sdk | `fetch('/chat', { method: 'POST', body: JSON.stringify({ message, sessionId }) })` |

**mastra の補足:**
- `@mastra/client-js` で Mastra 組み込みサーバーの Agent API（`/api/agents/task-agent/generate`）を直接呼ぶ
- `resourceId` + `threadId` で会話履歴が PostgreSQL に自動管理される
- カスタム `/chat` ルートは使わない

## インフラ

### フロントエンド Dockerfile（mastra-fastify / strands-* / claude-agent-sdk 共通パターン）

```dockerfile
FROM node:20-alpine
WORKDIR /app
RUN corepack enable && corepack prepare pnpm@latest --activate
COPY package.json pnpm-lock.yaml* ./
RUN pnpm install --frozen-lockfile
COPY . .
EXPOSE {port}
CMD ["pnpm", "dev", "--host", "0.0.0.0", "--port", "{port}"]
```

### Docker Compose 変更点

1. 各バックエンドサービスの `build` パスを `./mastra` → `./mastra/backend` に変更
2. 各フレームワークに `{framework}-frontend` サービスを追加（profile は同一）

```yaml
mastra-frontend:
  build: ./mastra/frontend
  ports:
    - "5001:5001"
  depends_on:
    - mastra
  profiles: [mastra]
```

### CORS

フロントエンドから Docker コンテナのバックエンドに直接 fetch するため、各バックエンドに CORS 設定を追加する。
Vite の proxy は使わない（比較学習として実際の URL を直接呼ぶ）。

## 実装順序

1. 各フレームワークの既存コードを `backend/` に移動（git mv）
2. docker-compose.yml の `build` パスを更新
3. 各バックエンドに CORS 設定を追加
4. 各フレームワークの `frontend/` を実装（mastra → 他4つ）
5. Makefile を更新（必要に応じて）
